循环路径说明：
1. correct_dsl → validate_dsl：DSL 修正后重新校验，最多 3 次重试
2. simplify_dsl → build_sql：执行失败后简化 DSL 重新生成 SQL

Agentic 节点：
• decompose：LLM 将复杂查询（对比/同比/趋势）改写为单 DSL 可表达形式
• correct_dsl：LLM 决策检索词 → 定向 RAG → 重生成（self-correcting RAG）
• verify_dsl：LLM 自检 DSL/结果是否真正回答了用户原始问题
