## ❌ 严重语义丢失（条件/过滤未识别）

### A1. 华东线上**金额大于5000**的订单
```
问题: "查询华东地区线上渠道金额大于5000的订单销售额"
❌ 丢失: "金额大于5000" → filters 中无 pay_amount > 5000
❌ 丢失: "订单" → 没有按订单维度统计，而是按 region+channel 汇总
```

### A2. **非手机品类**且**金额大于3000**
```
问题: "查询非手机品类且金额大于3000的销售额"
❌ 丢失: "非手机品类" → 无 category != '手机' 过滤，结果里还有手机
❌ 丢失: "金额大于3000" → 无 pay_amount > 3000 过滤
```

### A3. 苹果品牌**且金额在5000到20000之间**
```
问题: "查询苹果品牌产品在5000到20000元之间的销售额"
⚠️  部分: "苹果品牌" → 识别为 brand 维度（对），但未过滤 brand='苹果'
   结果里出现华为、海尔等非苹果品牌
❌ 丢失: "5000到20000元之间" → 无金额范围过滤
```

### G2. 超长复杂描述（最严重）
```
问题: "帮我查一下华东地区通过线上渠道购买的VIP客户中，单笔金额超过8000元的订单，
      按产品品牌分组统计销售额和订单数量，取前10名"

✅ 识别: 华东 → region_code='HD'
✅ 识别: 线上 → channel_code='online'
✅ 识别: VIP → customer_type='VIP'
✅ 识别: 品牌分组 → dimensions 包含 brand
⚠️  部分: 销售额 → metrics 有 sales_amount

❌ 丢失: "单笔金额超过8000元" → 无 pay_amount > 8000
❌ 丢失: "订单数量" → metrics 只有 sales_amount，无 order_count
❌ 错误: dimensions 冗余 → 包含了 product_name, region, channel, customer_type
         用户只要求 "按产品品牌分组"
❌ 错误: 结果只有1行（VIP+华东+线上 交集太小），但用户要求"取前10名"
```

## ❌ 隐含JOIN未触发

### D1. 按**供应商**统计销售额
```
问题: "查询各供应商的销售额"
❌ 错误: dimensions=['product_name']，完全没有 supplier_dim JOIN
   结果按产品名称分组，不是供应商
```

### D2. 按**客户名称**统计
```
问题: "查询各客户的消费金额"
⚠️  部分: 触发了 customer_dim JOIN（因为 keywords 匹配到"客户"）
   但 dimensions 是 customer_type 而不是 customer_name
```

### D3. 按**城市等级**统计
```
问题: "查询各城市等级的销售额"
❌ 错误: dimensions=['tier_level'] 是对的，但 SQL 中无 region_dim JOIN
   直接用了 order_fact.tier_level（该字段不存在，依赖 JOIN）
```

## ❌ 聚合/排名语义错误

### C3. 各品类**销售额最高的产品**
```
问题: "查询各品类中销售额最高的产品"
❌ 错误: dimensions=['category', 'product_name'] + ORDER BY sales_amount DESC
   这只是全局排序，不是"每个品类取第一"
   正确做法: 需要窗口函数 ROW_NUMBER() OVER (PARTITION BY category)
   或子查询: 先算每个产品的销售额，再按品类取 MAX
```

### E2. 各品类销售额占**总销售额的比例**
```
问题: "查询各品类销售额占总销售额的比例"
❌ 错误: 返回的是普通分组汇总，无占比计算
   正确做法: 需要子查询先算 total_sales，再算每个 category / total_sales
```

## 根因分析

当前系统使用 **关键词匹配的 Mock DSL 生成器**（无LLM时fallback），存在以下局限：

1. **无法处理复合条件**: "A且B且C" → 只能识别A，丢失B和C
2. **无法处理数值范围**: "大于5000"、"5000到20000" → 完全丢失
3. **无法处理否定**: "非手机" → 无法生成 != 过滤
4. **无法处理窗口函数**: "各品类最高" → 只能全局排序
5. **无法处理子查询/派生指标**: "占比" → 需要 total / group 的计算
6. **隐含JOIN依赖关键词**: "供应商"没有正确触发 supplier_dim JOIN
7. **时间语义缺失**: "1月份"、"最近" → 无时间过滤

## 改进方向

| 问题 | 解决方案 |
|------|----------|
| 复合条件丢失 | 接入LLM进行完整语义解析 |
| 数值范围 | DSL 支持 BETWEEN 操作符 |
| 否定条件 | 完善 Filter operator 支持 `!=`、`not in` |
| 窗口函数 | SQLBuilder 支持 ROW_NUMBER/RANK |
| 占比/子查询 | Aggregator 支持 proportion 计算 |
| 隐含JOIN | EntityResolver 增强维度→表映射 |
| 时间语义 | 添加时间解析器（相对时间→绝对时间） |
