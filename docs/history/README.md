## 归档规则

1. **实施完成后归档** — 只有当代码实现完毕、测试通过、PR 合并后，才将对应文档移入此目录
2. **保留原始结构** — 保持 `specs/` 和 `plans/` 子目录，便于追溯
3. **不删除只移动** — 历史文档永远保留，只是位置变更
4. **正式文档优先** — 任何冲突以 `docs/` 根目录下的正式文档为准
